# Jira Board Setup: SaaS Churn Dashboard

## Purpose
This document explains how the Jira Scrum board was structured for the Agile Analytics Delivery Project. The board is designed to track requirements, backlog items, sprint tasks, dashboard development, testing and final delivery.

## Jira Project Type
Scrum project

## Project Name
Agile Analytics Delivery Project

## Board Name
SaaS Churn Dashboard Delivery Board

## Issue Types Used

| Issue Type | Purpose |
|---|---|
| Epic | Major work areas such as requirements, data preparation, churn analysis and dashboard development. |
| Story | Stakeholder-focused requirements written from the user perspective. |
| Task | Specific delivery activities needed to complete a story or epic. |
| Bug | Issues found during dashboard testing or validation. |

## Jira Workflow Columns

| Column | Meaning |
|---|---|
| Backlog | Work identified but not yet selected for a sprint. |
| To Do | Work selected for the current sprint but not started. |
| In Progress | Work currently being completed. |
| In Review | Work ready for checking, feedback or validation. |
| Done | Work completed and accepted. |

## Epics to Create in Jira

| Epic ID | Epic Name |
|---|---|
| EPIC-01 | Requirements Discovery |
| EPIC-02 | Data Preparation |
| EPIC-03 | Churn Analysis |
| EPIC-04 | Revenue-at-Risk Analysis |
| EPIC-05 | Dashboard Development |
| EPIC-06 | Testing and Validation |
| EPIC-07 | Final Delivery and Recommendations |

## Sprint Setup

| Sprint | Sprint Name | Focus |
|---|---|---|
| Sprint 1 | Requirements and Planning | Stakeholder needs, scope, metrics and backlog. |
| Sprint 2 | Data Preparation and Churn Analysis | Dataset review, cleaning and churn analysis. |
| Sprint 3 | Revenue-at-Risk and Dashboard Build | Revenue-risk calculations and Power BI dashboard development. |
| Sprint 4 | Testing and Final Delivery | Validation, review, recommendations and retrospective. |

## Labels to Use

| Label | Purpose |
|---|---|
| requirements | Stakeholder needs and business requirements. |
| data-preparation | Data cleaning and preparation tasks. |
| churn-analysis | Churn metric and segment analysis tasks. |
| revenue-risk | Revenue-at-risk calculation tasks. |
| dashboard | Power BI dashboard design and build tasks. |
| validation | Testing and metric checking tasks. |
| recommendations | Final insights and business recommendations. |

## Priority Levels

| Priority | Meaning |
|---|---|
| High | Required for the dashboard to answer the core business problem. |
| Medium | Important for usability, documentation or stakeholder clarity. |
| Low | Useful enhancement, but not required for first delivery. |
