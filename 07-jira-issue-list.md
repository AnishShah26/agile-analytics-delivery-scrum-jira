# Jira Issue List: SaaS Churn Dashboard

## Purpose
This document lists the Jira issues created for the Agile Analytics Delivery Project. The issues are structured into epics, user stories and tasks to support Scrum-based delivery.

## Jira Project
Agile Analytics Delivery Project

## Jira Board
SaaS Churn Dashboard Delivery Board

## Issue Hierarchy
- Epics represent major delivery areas.
- User stories represent stakeholder needs.
- Tasks represent specific delivery activities.
- Sprints organise selected work into delivery cycles.

## Epics

| Epic ID | Epic Name | Description |
|---|---|---|
| EPIC-01 | Requirements Discovery | Define stakeholder needs, dashboard scope, success criteria and business questions. |
| EPIC-02 | Data Preparation | Prepare customer, subscription, usage, support and churn datasets for analysis. |
| EPIC-03 | Churn Analysis | Analyse churn rate and identify high-risk customer segments. |
| EPIC-04 | Revenue-at-Risk Analysis | Quantify monthly and annual recurring revenue exposure from churn. |
| EPIC-05 | Dashboard Development | Build an executive Power BI dashboard for stakeholder decision-making. |
| EPIC-06 | Testing and Validation | Validate dashboard metrics, filters and outputs before final delivery. |
| EPIC-07 | Final Delivery and Recommendations | Summarise findings, recommendations and delivery outcomes. |
